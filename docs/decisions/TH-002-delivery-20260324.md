## リスク・注意事項

1. **Google OAuth設定**: Supabase側のProvider設定とGoogle Cloud Consoleの設定が必要。手順を誤ると認証が動作しない
2. **RLSポリシー**: マイグレーション適用順序を守ること（schema → RLS の順）。RLS適用前にデータ移行を行うと権限エラーの可能性あり
3. **ISR初回アクセス**: デプロイ直後はキャッシュが空のため、初回アクセスが遅くなる可能性がある（Serverless Functions のコールドスタート）
4. **データ移行の完全性**: 移行スクリプト実行後、ダッシュボード上でTH-001のデータが正しく表示されることを目視確認すること
