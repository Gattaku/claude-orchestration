2026-03-25

## トリアージ結果
- Input種別: C（フィードバック/バグ）
- Input内容: 会議途中のデータがSupabaseに保存されない。AIPOがエージェントと議論する際、discussion_logsテーブルへの記録が行われていない。
- 規模: S
- パス: Express
- スキップするPhase: Phase 1, 2, 3, 4
- スキップ理由:
  - Phase 1: ユーザーリサーチ不要（バグ報告で原因が明確）
  - Phase 2: 価値定義不要（既存機能のバグ修正）
  - Phase 3: 簡易Storyで十分（下記参照）
  - Phase 4: 新技術不要（Claude Code hooks + 既存Supabase API）

## 影響範囲（AI Dev調査結果）
- 影響するファイル:
  - `.claude/hooks/log-discussion.mjs`（新規作成）
  - `.claude/settings.json`（新規作成 - hooks設定）
  - `.claude/agents/aipo.md`（ルール更新）
  - `docs/decisions/theme-registry.md`（テーマ登録）
- 技術的リスク:
  - hookスクリプト失敗時にClaude Code全体への影響（exit 0で常に正常終了するよう対策済み）
  - プロンプトにTH-XXXが含まれない場合は記録されない（ルールで明示し対策）
  - メッセージが長大になる可能性あり（3000文字で切り詰め対策済み）

## バグの根本原因分析
1. `scripts/insert-discussion-log.ts` CLIスクリプトは存在し正常動作する
2. AIPO定義ファイルに「Agent呼び出し前後にスクリプトを手動実行せよ」と指示があるが、強制力がない
3. 結果として、AIPOはほぼ100%の確率でこのコマンド実行をスキップしていた
4. つまりdiscussion_logsテーブルへのINSERTが一切行われていなかった

## 修正方針
Claude Codeの `PostToolUse` hook機能を利用し、Agentツール使用後に自動でdiscussion_logsへINSERTする仕組みを構築。

### 検討した選択肢

#### 案A: PostToolUse hookによる自動記録（採用）
- メリット: システムレベルで強制実行、記録漏れなし、request/response両方を自動記録
- デメリット: メッセージが要約されず全文が入る（3000文字で切り詰めで対応）

#### 案B: サブエージェント自己記録（不採用、フォールバックとして維持）
- メリット: 実装コスト低、要約された記録が残る
- デメリット: LLMがプロンプト指示に従うか不確実（現にバグの原因）

#### 案C: hooks + サブエージェント自己記録のハイブリッド（採用）
- 案Aを主軸とし、案Bは既存のフォールバック指示を維持する方針で採用

### 却下理由
- 案Bのみでは解決にならない（現にバグの原因が「指示に従わない」ことであるため）

## 簡易Story
- タイトル: 議論ログのSupabase自動Push
- 目的: AIPOがAIエージェントと議論する際、議論内容が自動的にSupabaseに記録されるようにする
- 受け入れ条件:
  - [ ] Agent(ai-pm/ai-pd/ai-dev)呼び出し後、discussion_logsテーブルにrequest/responseの2件が自動INSERTされる
  - [ ] プロンプトにTH-XXXが含まれない場合は静かにスキップされる（エラーにならない）
  - [ ] Supabase環境変数が未設定でもClaude Code全体が停止しない
  - [ ] hookスクリプトの実行エラーがClaude Codeの会話フローを妨げない
