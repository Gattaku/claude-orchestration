# Phase 0: トリアージ — 詳細展開パネルの閉じるボタンUX改善

## 日付
2026-03-25

## トリアージ結果
- Input種別: C（フィードバック / UX改善要望）
- Input内容: 「詳細を見る」で展開した後、閉じるためにページ上部までスクロールしなければならず使いづらい
- 規模: S
- パス: Express
- スキップするPhase: Phase 1, 2, 3, 4
- スキップ理由:
  - Phase 1: ユーザーリサーチInputではない
  - Phase 2: 課題と解決策が明確（既存UIの使い勝手改善）
  - Phase 3: 簡易Storyで十分な規模
  - Phase 4: 新技術導入不要、既存コンポーネント内の修正のみ

## 影響範囲（AI Dev調査結果）
- 影響するファイル: `src/components/timeline-entry.tsx` のみ
- 技術的リスク: 極めて低い。JSXの追加のみでstate管理やprops変更は不要

## 検討したアプローチ

### 案A: 展開セクション下部にも「閉じる」ボタンを追加（採用）
- メリット: 最もシンプル、実装コスト低、既存UIパターンとの一貫性
- デメリット: ボタンが2箇所に分散

### 案B: ボタンのsticky化（却下）
- メリット: 常にボタンが見える
- デメリット: max-height + overflow-y の追加が必要。展開エリア内にスクロールバーが出てモバイルで扱いづらい

### 案C: scrollIntoView で上部に戻す（却下）
- メリット: ボタンは1箇所のまま
- デメリット: 閉じた後にページ位置が飛ぶ、UXとして不自然

## 簡易Story
- タイトル: 詳細展開パネル下部に「閉じる」ボタンを追加
- 目的: 長い詳細コンテンツを展開した後、上までスクロールせずに閉じられるようにする
- 受け入れ条件:
  - [ ] 展開されたbody_htmlセクションの下部に「閉じる」ボタンが表示される
  - [ ] 下部の閉じるボタンをクリックすると、展開セクションが閉じる
  - [ ] 上部の既存ボタンは従来通り動作する
  - [ ] ダークモードでも正しく表示される
