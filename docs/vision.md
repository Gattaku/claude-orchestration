# AI Orchestration Vision

## 背景

フルスタックWebアプリ開発のデベロッパーとして、アジャイル開発・バランスチーム体制で働いている。
現在の業務フローをAI駆動開発に移行し、ロールの垣根を超えた新しい開発体制を構築したい。

## 現在の開発体制

- **アジャイル開発** / **バランスチーム**（PM・PD・Dev）
- **ユーザー中心設計**: ユーザーインタビューからインサイトを得て、プロダクトの方向性を決める
- **D&F（Discovery & Framing）**: 発見と発散を重視し、収束させるプロセス
- **MVP思考**: 小さく始めて大きく育てる
- **TDD**: テスト駆動開発を主体とし、テスト → 実装 → リファクタのサイクルを回す
- **Story管理**: GitHub Issueで管理し、1つの価値を単位としたStoryベースで開発を進める

## AI駆動開発のビジョン

### 核心的な考え方

- コードを書けること自体の価値はなくなっていく
- 「どんな価値を創出するか」に価値がシフトし、その意思決定だけを人間が持つ
- ロールの垣根はなくなり、一人の人間がAIチームを統括する形になる

### 体制構造

```
人間（オーナー / レフェリー）
  ├── ユーザーインタビュー・生の情報収集（議事録をmdで提供）
  ├── AIの議論プロセスの監視・軌道修正
  ├── 最終的なGo/No-Goの判断
  │
  └── AIPO（AIプロダクトオーナー / オーケストレーター）
        ├── AI PM × 複数  ... 議論・発散・Story策定
        ├── AI PD × 複数  ... UX/UI議論・デザイン提案
        └── AI Dev × 複数 ... TDD実装・コードレビュー・デプロイ
```

### 人間の役割

- **プレイヤーではなくレフェリー兼オーナー**
- ユーザーインタビューの議事録（mdファイル）をInputとして提供
- AIチームの議論プロセスを監視し、方向がおかしいときだけ介入
- 各フェーズの区切りでGo/No-Goの判断を下す

### AIオーケストレーションの流れ

1. **Input**: 議事録のmdファイル（人間が提供）
2. **AIPO** が議事録を読み、AI PM・AI PD・AI Dev と議論を開始
3. **インサイト抽出** → 価値の定義 → ロードマップとの整合
4. **Story策定** → UX/UI設計 → 実装・テスト・デプロイ
5. 各フェーズの区切りで人間にレビュー・Go/No-Goを求める
6. **Output**: 動くプロダクト

## 最初に作るプロダクト: AI開発管理アプリ

### 概要

このAIオーケストレーション自体の動きを可視化するWebアプリケーション。

### 必要な画面・機能

- AIエージェント間の**議論内容**が見える
- 議論の**結果**が見える
- 結果から生まれた**Story**が見える
- AI Devの**開発進捗**が見える
- アプリ全体の**開発状況**が確認できる

### 技術スタック

- Vercel + Next.js + Supabase を基本候補とするが、AIチームの議論で最終決定する

## 開発アプローチ

1. まず各AIロール（AIPO, AIPM, AIPD, AIDev）のカスタムプロンプトを作成
2. 一つずつ動きを確認しながら調整
3. 動作確認後、最終的なエージェントシステムとして統合
